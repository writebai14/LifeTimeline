# LifeTimeline 开发任务清单

> 按实现顺序拆分的可执行任务，对应 PRD 中的 P0/P1。

---

## 阶段一：数据与存储（先做）

- [ ] **1.1** 定义并实现本地存储方案
  - 选项 A：按日 JSON 文件（如 `data/2025-12-18.json`）
  - 选项 B：SQLite + 媒体文件目录
  - 确定应用数据目录位置（如 `~/Library/Application Support/LifeTimeline` 或项目内 `./data`）
- [ ] **1.2** 实现 Day / Block / Media 的读写 API（或直接读写 JSON）
- [ ] **1.3** 媒体文件存储：拍照/截图保存到固定目录，DB/JSON 只存路径与元数据

---

## 阶段二：日视图与时间块（核心）

- [ ] **2.1** 日视图页面：左侧时间轴（0:00–24:00，15 分钟一格）
- [ ] **2.2** 时间块在时间轴上的展示（起止对应刻度，可折叠/展开）
- [ ] **2.3** 时间块创建：点击空白处或按钮，弹窗/内联编辑（起止、摘要、分类、地点、备注）
- [ ] **2.4** 时间块编辑与删除
- [ ] **2.5** 时间块合并：选两个相邻块合并为一个
- [ ] **2.6** 日期选择器：切换日期后加载并展示该日数据
- [ ] **2.7** 默认进入“今天”

---

## 阶段三：低成本输入（对齐场景 1）

- [ ] **3.1** “一句话”入口：输入一句话 → 以当前时间为起点生成一个块（默认 15 或 30 分钟），可事后改时间与分类
- [ ] **3.2** “拍照/截图”入口：选择文件或调起相机/截图 → 保存文件 + 写入 Media（capturedAt=当前时间），在日视图中显示为“待补记素材”
- [ ] **3.3** 待补记素材在日视图中的展示：按时间排列的缩略图/列表
- [ ] **3.4** 将某条素材“关联到已有块”或“生成新块”：编辑块时选择关联媒体；或从素材点“生成块”并填起止与分类

---

## 阶段四：分类与元数据

- [ ] **4.1** 分类标签管理：预设 + 用户自定义，存为列表或表
- [ ] **4.2** 时间块编辑时选择分类、地点（可下拉+可输入）、备注
- [ ] **4.3** （可选）情绪/天气字段：枚举或短文本

---

## 阶段五：任务区与日总结（P1）

- [ ] **5.1** 当日“今日任务/明日目标/本周任务”编辑区（可折叠）
- [ ] **5.2** 当日“今日完成/未完成/超额完成”编辑区
- [ ] **5.3** 与时间块同屏展示（如日视图顶部或侧栏）

---

## 阶段六：检索与统计（P1）

- [ ] **6.1** 按关键词或标签检索：返回匹配的日期列表或块列表
- [ ] **6.2** 按日期范围筛选日志
- [ ] **6.3** 统计页：按分类汇总时间（日/周/月/自定义范围），简单图表或表格
- [ ] **6.4** 导出：选定日期或范围导出为 Markdown 或 JSON

---

## 阶段七：增强（P2）

- [ ] **7.1** 情绪/状态趋势图（若已记录情绪）
- [ ] **7.2** 智能建议（仅建议）：如根据照片时间建议插入块，用户确认后才写入
- [ ] **7.3** 从现有备忘录文本解析导入（可选）

---

## 建议技术栈（可替换）

| 层级 | 建议 | 备选 |
|------|------|------|
| 桌面壳 | Tauri | Electron |
| 前端 | React + TypeScript | Vue / Svelte |
| 状态/数据 | 直接读写在本地 JSON 或 SQLite | Zustand/Redux + 本地 API |
| 存储 | SQLite (better-sqlite3 / sql.js) 或 按日 JSON | 同左 |
| UI | Tailwind + 简单组件库 | 纯手写 CSS |

---

按上述顺序做完阶段一～四即可覆盖“记录成本低于备忘录”的日视图与快速输入；阶段五、六覆盖复盘与回忆场景。
