# LifeTimeline 开发需求文档

> 本地个人时间轴日志系统 · 产品需求与开发规格  
> 版本：v0.1 · 仅供个人使用、本地部署

---

## 一、产品概述

### 1.1 产品名称
**LifeTimeline**（本地个人时间轴日志系统）

### 1.2 产品目标
以 **15 分钟** 为最小颗粒度，在本地记录每日时间使用、状态、想法与事件，支持：
- 检索与复盘（某天/某时段做了什么）
- 统计（时间分配、情绪趋势）
- 回忆（记账时按日期回溯、人生决策复盘）
- 照片/截图与时间轴关联，便于事后补记与回顾

### 1.3 核心原则（开发必须遵守）

| 原则 | 说明 | 对开发的影响 |
|------|------|----------------|
| **数据 100% 本地** | 无云端同步、无账号、无外网依赖 | 存储用本地文件/本地 DB；可选导出备份 |
| **自动化只建议不决定** | AI/智能只能推荐，最终由用户确认或修改 | 如有“智能建议”，必须显式展示并需用户采纳 |
| **记录成本 < 苹果备忘录** | 输入要更少、更快、更省心 | 优先快捷输入、模板、照片驱动补记 |
| **日视图是核心** | 其他视图（周/月/统计）均为辅助 | 日视图优先实现并打磨；其余可后续迭代 |
| **15 分钟为逻辑单位，允许合并** | 一个块可跨多个 15 分钟 | 时间块可拖拽调整起止，支持“合并/拆分”语义 |

## 二、用户场景与需求映射

### 场景 1：白天忙碌
**行为：** 随手拍照、偶尔写一句话、不做完整记录。

| 需求编号 | 需求描述 | 优先级 |
|----------|----------|--------|
| R1.1 | 支持「快速拍照/截图」并自动带时间（可选地点），先存为“待补记素材” | P0 |
| R1.2 | 支持「一句话速记」：当前时间 + 一句话，自动生成一个时间块草稿 | P0 |
| R1.3 | 单次操作步骤 ≤ 2 步（如：点一下“拍照”或“一句话”即可） | P0 |

### 场景 2：晚上补记
**行为：** 打开当天 → 看照片 → 根据记忆补时间段 → 填分类/状态/备注。

| 需求编号 | 需求描述 | 优先级 |
|----------|----------|--------|
| R2.1 | 日视图按时间轴展示当天所有“块”（含未分类的素材与草稿） | P0 |
| R2.2 | 支持按 15 分钟刻度划分，可点击/拖拽创建、编辑、合并时间块 | P0 |
| R2.3 | 时间块可设置：起止时间、分类标签（如【工作】【摸鱼】【WOW】）、地点、备注、情绪/天气等 | P0 |
| R2.4 | “待补记素材”在日视图中按时间排列，支持将某张照片/截图关联到某个时间块或生成新块 | P0 |
| R2.5 | 提供今日任务/明日目标/本周任务等“清单区”（可折叠），与时间块可关联或独立 | P1 |

### 场景 3：记账回忆
**行为：** 选某一天 → 快速看到那天做了什么。

| 需求编号 | 需求描述 | 优先级 |
|----------|----------|--------|
| R3.1 | 支持按日期快速跳转（日期选择器/日历点选） | P0 |
| R3.2 | 某日视图可一览：时间轴 + 当日任务/完成情况（若有） | P0 |
| R3.3 | 支持按关键词/标签检索“某几天”的日志（列表或简略时间轴） | P1 |

### 场景 4：人生复盘
**行为：** 查某个时期 → 看情绪变化、看时间分配。

| 需求编号 | 需求描述 | 优先级 |
|----------|----------|--------|
| R4.1 | 支持按日期范围筛选日志 | P1 |
| R4.2 | 统计视图：按分类的时间分配（日/周/月/自定义范围） | P1 |
| R4.3 | 若记录情绪/状态，支持简单趋势或分布展示 | P2 |
| R4.4 | 导出为可读格式（如 Markdown/JSON）便于备份与二次分析 | P1 |

## 三、数据模型（建议）

### 3.1 核心实体

```
日期 (Day)
├── date: 日期 (YYYY-MM-DD)
├── 今日任务 / 明日目标 / 本周任务 (可选，文本或结构化)
├── 日总结：今日完成 / 未完成 / 超额完成 (可选)
└── 时间块列表: [Block]

时间块 (Block)
├── id
├── start, end: 开始/结束时间 (精确到分钟，建议步长 15)
├── summary: 简短描述 (如 "驾车通勤")
├── category: 分类标签 (如 工作/摸鱼/研究/WOW/梦幻)
├── location: 地点 (如 公司、张江社区蔚来充电站)
├── note: 备注 (长文本)
├── mood/weather: 情绪或天气 (可选，可枚举或自由文本)
├── linkedMediaIds: 关联的媒体 id 列表
└── 来源: 手动创建 | 从素材创建 | 从一句话速记创建

媒体/素材 (Media)
├── id
├── capturedAt: 拍摄/截图时间
├── filePath: 本地路径（或相对路径）
├── type: photo | screenshot
├── optionalLocation: 可选地理位置
└── linkedBlockId: 关联到哪个 Block（可选，未关联则出现在“待补记”）
```

### 3.2 分类标签
- 支持用户自定义标签（如【工作】【摸鱼】【研究】【WOW】【梦幻】）。
- 可预设若干常用标签，其余用户自增。
- 标签可在统计中聚合。

### 3.3 存储
- **本地文件 + 本地数据库** 二选一或组合：
  - 方案 A：每日期一个文件（如 JSON/Markdown），便于版本/备份。
  - 方案 B：SQLite（或类似）存 Block/Media/标签，便于查询与统计。
- 照片/截图建议单独目录存储，数据库只存路径与元数据。

## 四、功能清单与优先级

### P0（首版必须）
- [ ] **日视图**：按时间轴展示当日时间块与待补记素材；支持 15 分钟刻度。
- [ ] **时间块 CRUD**：创建、编辑、删除、合并；起止时间、摘要、分类、地点、备注。
- [ ] **快速输入**：一句话速记（当前时间 + 文本）；快速拍照/截图入库并带时间。
- [ ] **待补记素材**：按时间展示未关联的照片/截图；支持关联到已有块或生成新块。
- [ ] **日期切换**：选日期后展示该日日视图。
- [ ] **数据 100% 本地**：存储与读取均不依赖网络。

### P1（第二波）
- [ ] 今日任务/明日目标/本周任务 与 日总结（今日完成/未完成/超额完成）。
- [ ] 按关键词或标签检索日志（某几天列表或简略时间轴）。
- [ ] 按日期范围筛选。
- [ ] 按分类的时间分配统计（日/周/月/自定义范围）。
- [ ] 导出（Markdown 或 JSON）。

### P2（后续增强）
- [ ] 情绪/状态趋势或分布图。
- [ ] 更多统计维度与可视化。
- [ ] 若需“智能建议”：仅做建议（如根据照片时间建议块），且需用户确认后才写入。

## 五、UI/UX 要点

### 5.1 日视图（核心）
- 左侧或顶部：时间轴（0:00–24:00，步长 15 分钟）。
- 右侧/主体：每个时间块占一行或一段，可折叠为“摘要”或展开为“详情”。
- 待补记素材以缩略图形式嵌在对应时间附近，可拖拽到某块或点击“生成新块”。
- 支持空白处点击/拖拽创建新块，块可拖拽调整起止。

### 5.2 记录成本 < 备忘录
- 入口明显：打开应用默认进入“今天”；提供“一句话”“拍照”快捷按钮（常驻或快捷键）。
- 默认值：新建块默认“当前时间”开始、15 分钟或 30 分钟长度；分类可记忆上次选择。
- 模板：可选“通勤”“会议”“休息”等模板一键生成块再微调。

### 5.3 与现有格式的对应
- 您当前的备忘录格式可直接对应到：**日期 + 任务区 + 时间块列表 + 日总结**。
- 导入（P2 或更晚）：若需从现有备忘录迁移，可做“解析一段文本 → 生成 Day + Blocks”的导入工具。

## 六、非功能需求

| 类型 | 要求 |
|------|------|
| **平台** | 先支持 macOS（您当前以电脑为主）；若需移动端补记，可后续考虑 iOS/Android 或 PWA。 |
| **性能** | 单日渲染流畅；按日期加载，不一次性加载全部历史。 |
| **备份** | 数据目录可打包备份；导出功能可作为二次备份。 |
| **隐私** | 无埋点、无上报、无账号；可选本地加密（P2）。 |

## 七、技术选型建议（仅供参考）

- **桌面端**：Electron 或 Tauri + 本地 SQLite/JSON 文件。
- **前端**：React/Vue/Svelte 均可；日视图可用虚拟列表处理长轴。
- **存储**：SQLite（便于查询与统计） + 媒体文件存目录；或纯 JSON 按日存储（实现简单，查询靠内存或简单索引）。
- **照片/截图**：选择文件或调用系统相机/截图，存到应用数据目录并写入 Media 表。
