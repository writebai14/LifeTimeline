# 开发者接手指南

## 1. 项目当前状态（2026-03-05）
- 核心流程已可用：按天记录时间块、批量上传图片、导入导出文本、日期切换。
- 新增了“记录活跃度日历”（格子 + 颜色深浅），放在标题栏按钮内的弹出层，默认展示最近 14 周。
- 左侧时间轴样式已强化分类可视化，不同 tag 的块区分明显。

## 2. 代码地图（先读这些文件）
- `src/App.tsx`：应用入口状态管理；日期切换、保存逻辑、活跃度弹层接入。
- `src/DayView.tsx`：单日主界面；时间轴渲染、时间块编辑入口、导入导出。
- `src/DayView.css`：时间轴视觉样式（重点看 `.timeline-*` 与 `.block-card*`）。
- `src/ContributionCalendar.tsx`：格子日历逻辑（时间范围、分级阈值、点击跳转）。
- `src/api.ts` + `server/index.js`：前后端数据接口。
- `src/types.ts`：数据模型定义（`Day` / `Block` / `Media`）。

## 3. 数据与存储
- 日志数据：`data/days/YYYY-MM-DD.json`
- 媒体文件与元数据：`data/media/*`（图片 + 同名 `.json`）
- 开发环境代理：
  - 前端 `5173`
  - 后端 `3001`
  - 代理配置见 `vite.config.ts`

## 4. 活跃度日历维护说明
- 分数函数在 `src/App.tsx` 的 `dayRichnessScore`。
- 颜色分级在 `src/ContributionCalendar.tsx` 的 `LEVEL_COLORS` 与 `scoreThresholds`。
- 交互入口在标题栏 `.header-actions`，组件默认显示 `weeksToShow=14`。
- 如要改“什么算丰富”：优先只改 `dayRichnessScore`，保持 UI 组件纯展示。

## 5. 后续开发建议（低风险高收益）
- 增加“活跃度计算说明”悬浮提示，解释颜色含义与分值来源。
- 增加“仅显示有记录日期”筛选，减少空白格密度。
- 增加“周/月统计总览”，与格子图联动。

## 6. 协作约定（建议执行）
- 每次功能修改后，必须同步更新：
  - `docs/CHANGELOG.md`（写清日期、功能、影响文件）
  - 若涉及架构/入口变化，同步更新本指南
- PR 模板建议固定三段：
  - 变更目标
  - 影响范围（文件）
  - 验证方式（命令 + 人工检查点）
